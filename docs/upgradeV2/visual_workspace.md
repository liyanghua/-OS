三块核心底座：

1. **模板中心**：沉淀各类内容模板
2. **编译台**：承接用户 prompt、商品信息、品牌信息、参考图，生成结构化计划
3. **AI-native 视觉工作台**：做最终的对象化编辑、变体、确认与沉淀

所以现在不是再加功能，而是要把三者之间的**衔接逻辑、页面信息架构、用户心智路径**做顺。

我建议把“视觉工作台”从一个单纯改图页面，升级成：

# 一、重新定义：视觉工作台不是画布，而是“结果装配台”

它不是给用户从零创作的。
它的职责应该是：

> **把用户意图 + 已匹配模板 + 编译产物 + 当前视觉对象，装配成一个可编辑、可分支、可确认的结果空间。**

所以工作台的目标不是“自由”，而是：

* 快速看到 AI 已经理解了什么
* 快速看到 AI 准备生成什么结果
* 快速对关键对象进行自然语言修改
* 快速比较不同方案
* 快速确认“这版可以用了”

这决定了页面必须以“结果”为中心，而不是以工具为中心。

---

# 二、建议你把视觉工作台拆成一个统一的三段式链路

不是三页割裂，而是在工作台里把上游上下文显性化：

## 1）意图层

用户到底要什么

## 2）编译层

系统准备怎么做

## 3）结果层

当前实际生成和编辑的内容是什么

这三层要同时在页面里被看到，而不是藏在不同系统里。

---

# 三、建议的核心设计：一屏四区，而不是传统三栏

你前面的三栏适合纯编辑。
但你现在要接上模板中心和编译台，最好升级成：

```text
┌────────────────────────────────────────────────────────────────────┐
│ 顶部：任务标题 / 输出类型 / 意图摘要 / 模板匹配 / 一键生成 / 发布      │
├──────────────┬──────────────────────────────┬──────────────────────┤
│ 左侧：上游上下文 │ 中间：结果工作区                │ 右侧：AI编辑与执行区   │
│               │                              │                      │
│ 1. 用户意图     │ 1. 当前结果（主图/详情/视频）      │ 1. 当前选中对象上下文    │
│ 2. 模板与策略     │ 2. 画布 / 模块 / 分镜             │ 2. AI建议动作           │
│ 3. 编译计划      │ 3. 结果版本/变体切换              │ 3. 对话编辑器           │
│ 4. 结果结构树     │ 4. 对比 / 预览                   │ 4. 执行提案 / 风险提示   │
└──────────────┴──────────────────────────────┴──────────────────────┘
│ 底部：版本时间线 / 通过状态 / 资产沉淀 / 下一步动作                    │
└────────────────────────────────────────────────────────────────────┘
```

这比传统画布更适合你的业务。

# 四、核心原则：用户不应该“自己拼”，而应该“看 AI 装配，再局部干预”

你的系统不是设计师工具，而是操盘/内容团队工具。
所以用户理想路径应该是：

### Step 1

输入一句需求，或从编译台进入

### Step 2

系统自动展示：

* 我识别到你的意图
* 我匹配了哪些模板
* 我准备生成哪些模块/图/镜头

### Step 3

用户直接看到初版结果

### Step 4

用户只做少量干预：

* 换模板
* 改模块
* 改对象
* 改文案
* 改风格
* 出变体

### Step 5

确认、沉淀、流转

所以视觉工作台要像一个**“AI 已经先干了 80%”** 的界面。

# 五、重新组织页面：把“用户意图、模板、策略对象、编辑对象”显性串起来

---

## A. 顶部区域：任务总控条

这个区域一定要非常强，因为它负责建立全局心智。

建议包含：

### 1）当前任务

* 任务名：氨基酸洗面奶主图生成
* 输出类型：主图 / 详情页 / 视频脚本 / 买家秀
* 当前模式：编辑中 / 待确认 / 已通过

### 2）意图摘要

例如：

* 面向 18-29 岁女性
* 需要 5 张主图
* 风格偏真实生活感
* 重点突出温和不刺激

### 3）模板匹配结果

例如：

* 已应用模板：主图 5 张模板 v2
* 品牌策略：通用日化详情模板
* 视觉参考：竞品拆解 schema A

### 4）全局动作按钮

* 重新编译
* 更换模板
* 批量出变体
* 进入对比模式
* 发布到资产/计划板

这块的作用是：
**用户永远知道当前结果是从哪来的。**

---

## B. 左栏：从“素材栏”升级为“上下文与结构栏”

左栏不要只是对象树。
建议变成 4 个 tab。

---

### Tab 1：用户意图

展示编译台来的结构化输入

```text
商品：氨基酸洗面奶
目标：生成 5 张主图
人群：18-29 女性
风格：都市白领公寓 / 真实生活感
必须呈现：产品特写 / 使用状态 / 全景环境
```

这里支持：

* 编辑意图
* 追加约束
* 刷新编译

作用：用户如果发现系统理解偏了，可以在这里改，而不是进聊天反复说。

---

### Tab 2：模板与策略

展示：

* 当前主模板
* 当前品牌模板
* 当前子模板/模块模板
* 核心卖点
* 必须包含
* 禁止动作
* 品牌规则

例如：

* 主图模板：5 张主图叙事模板
* 当前图位：第 2 张痛点对比图
* 品牌规则：商品颜色不可偏移，logo 区不可遮挡
* 当前卖点：温和清洁 / 洗后不紧绷

这里支持：

* 替换模板
* 只替换当前模块模板
* 锁定策略

---

### Tab 3：编译计划

展示系统怎么从“意图 + 模板”得到当前结果

例如：

* 输出计划：

  * 图 1：人群场景直击
  * 图 2：痛点对比
  * 图 3：核心需求满足
  * 图 4：功能对比
  * 图 5：信任背书

或者详情模式：

* 模块 1：吸引注意
* 模块 2：独特设计
* 模块 3：特殊材料

或者视频模式：

* 镜头 1：强痛点开场
* 镜头 2：对比反差
* 镜头 3：核心卖点响应

这会让用户觉得系统是“有计划”的，不是随机生成。

---

### Tab 4：结果结构树

这里才放具体编辑对象。

按当前模式显示不同结构：

#### 主图模式

* 图 1
* 图 2
* 图 3
* 图 4
* 图 5

#### 详情模式

* 模块 1
* 模块 2
* 模块 3
* …

#### 视频模式

* 镜头 1
* 镜头 2
* 镜头 3
* …

每个节点展开后再看到：

* 对象树
* 文案区
* 背景区
* 主体商品
* 人物
* 装饰元素

这就是“脚本树 + 对象树”的组合。

# 六、中间主区：不是单一画布，而是“结果工作区”

这一块是你的核心差异。

建议按输出类型切 3 种工作模式，但交互保持一致。

---

## 模式 1：主图模式

中间显示：

* 当前 5 张图的缩略序列
* 选中某一张后进入大画布
* 大画布里支持对象选中、区域框选、对话改图
* 下方显示同图位变体

用户心智：
“我在编辑第 2 张主图，不是随便改一张图。”

---

## 模式 2：详情模式

中间显示：

* 详情页模块长页结构
* 每个模块可以点击进入局部编辑
* 支持“模块级画布”
* 也支持全页预览模式

用户心智：
“我在编辑模块 3 的视觉与文案，不是编辑一张散图。”

---

## 模式 3：视频模式

中间显示：

* 镜头卡片流
* 当前镜头 storyboard
* 镜头对象和口播联动
* 可切 shot list / 分镜图 / 文案表

用户心智：
“我在编辑镜头 5 的构图和口播，不是编辑一个孤立静态帧。”

# 七、右栏：不要只是对话框，要变成“AI 执行控制台”

右栏是用户和 AI 协作的核心，建议分 4 段。

---

## 1）当前上下文卡

明确告诉用户现在在改什么：

* 当前任务：主图生成
* 当前模块：第 3 张 核心需求满足图
* 当前对象：Hero Product
* 当前约束：不可改色 / 可改背景 / 可增强突出度

这一步非常重要，避免用户一句“放大一点”结果 AI 放错对象。

---

## 2）建议动作区

基于当前：

* 用户意图
* 当前模板
* 当前模块
* 当前评估结果

给出系统建议。

比如在第 2 张“痛点对比图”里建议：

* 强化左右差异
* 增加局部标注
* 缩短标题
* 提高产品占比

在第 1 张“场景直击图”里建议：

* 加强人物与产品关系
* 背景更生活化
* 降低元素杂乱度

这里的 chips 不再通用，而是**模块感知的建议**。

---

## 3）对话编辑器

这里建议不是一个纯聊天框，而是一个“上下文增强输入框”。

输入框上方显示：

* 当前目标
* 当前模板
* 当前对象

用户输入一句话时，系统自动带入这些上下文。

例如用户说：
“右边更柔和一点，文案更短”

系统实际解释为：

* 当前图位：第 2 张痛点对比图
* 右边 = 本品使用后区域
* 柔和 = 视觉对比强度降低一点
* 文案更短 = 标题文案压缩

这就是真 AI-native。

---

## 4）执行提案区

AI 不应直接改，而应先生成提案卡：

* 理解摘要
* 将修改的对象
* 修改步骤
* 预期效果
* 风险提示
* 按钮：执行 / 分步执行 / 只生成变体 / 取消

这是商用系统里很必要的安全层。

---

# 八、最关键的统一机制：所有内容都围绕“当前结果节点”工作

你现在最需要一个统一抽象，不然主图、详情、视频会散。

我建议统一成：

## `ResultNode`

```yaml id="h34deu"
ResultNode:
  id: string
  result_type: main_image | detail_module | video_shot
  parent_plan_id: string
  template_ref: string
  strategy_ref: string
  intent_ref: string
  title: string
  objective: string
  current_asset_ref: string
  object_tree_ref: string
  status: draft | editing | reviewed | approved
```

这样：

* 主图第 1 张，是一个 ResultNode
* 详情页模块 3，是一个 ResultNode
* 视频镜头 5，也是一个 ResultNode

视觉工作台本质上就是在编辑某个 ResultNode。

这会让整个平台特别统一。

# 九、最推荐的产品交互链路

我给你一个更顺的用户路径。

---

## 路径 1：从编译台进入工作台

### 第一步：输入需求

用户在编译台输入：

* 商品
* 人群
* 输出类型
* 参考风格
* 数量/模块要求

### 第二步：系统自动匹配

匹配：

* 主模板
* 品牌模板
* 参考 schema
* 输出计划

### 第三步：进入视觉工作台

不是空白进入，而是带着：

* 意图摘要
* 模板摘要
* 编译计划
* 初版结果

### 第四步：局部优化

用户只做关键干预。

---

## 路径 2：从模板中心进入工作台

适合更强操作型用户。

用户先选：

再带着模板进入编译和工作台。
这适合你已有模板中心的路径。

---

## 路径 3：从已有结果反向进入工作台

适合复用和优化。

用户直接打开一个历史主图/详情方案/视频脚本，系统自动加载：

* 它来自哪个模板
* 它基于什么意图
* 当前对象树和版本树

然后继续编辑。

# 十、视觉工作台最值得加的 5 个强化点

---

## 1）“当前结果为什么长这样”解释层

让系统解释：

* 这张图应用了哪个模板
* 哪些元素来自用户意图
* 哪些来自品牌规则
* 哪些来自竞品参考

这会极大提高用户信任。

---

## 2）“快速切换模板但保留结果”能力

比如当前第 1 张主图已经生成了，用户想把模板从“场景直击”换成“产品特写优先”。

不要全重来，而是：

* 保留商品与场景对象
* 只重编排结构
* 生成新分支

这很强。

---

## 3）“按模块出变体”而不是整案乱出

例如：

* 只给第 2 张出 4 个对比方案
* 只给模块 3 出 3 个高级感版本
* 只给镜头 5 出两种运镜方案

这比整案重跑高效太多。

---

## 4）“对象修改和脚本修改联动”

比如用户把第 3 张主图的场景从洗手台改成浴室镜前，系统应同步提示：

* 文案是否要更新
* 当前模块目标是否仍然匹配
* 是否建议切换为更合适的子模板

这就是你平台相对纯 AIGC 工具的优势。

---

## 5）“最终结果卡”输出

每个结果节点编辑完成后，输出一张标准化结果卡：

* 来源意图
* 应用模板
* 当前卖点
* 当前版本
* 评分
* 是否推荐
* 是否沉淀为模板样例

便于后面接资产图谱和 Plan Board。
