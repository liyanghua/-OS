为每个类目建立一套 **Category Insight Lens / 类目洞察透镜**。

---

# 一、桌布类目和假发类目的机会逻辑完全不同

## 1. 桌布类目：偏“空间场景型消费”

桌布用户的核心问题通常是：

> 我想让一个空间、一个饭桌、一个拍照场景变好看。

所以机会卡主要看：

| 维度   | 桌布类目关注点                 |
| ---- | ----------------------- |
| 场景   | 餐桌布置、出租屋改造、节日布置、野餐、拍照背景 |
| 风格   | 法式、奶油风、复古、田园、ins风、中古风   |
| 功能   | 防水、防油、易清洗、防烫、垂感、耐脏      |
| 人群   | 独居女生、宝妈、租房党、民宿主、小餐饮店主   |
| 内容爆点 | 改造前后对比、低成本氛围感、节日仪式感     |
| 商品机会 | 新图案、新材质、新尺寸、新套装组合       |

桌布类目机会卡的本质是：

> **某类人，在某个空间/节日/生活场景下，需要一种低成本提升氛围感的解决方案。**

---

## 2. 假发类目：偏“身份变化 + 信任决策型消费”

假发用户的核心问题通常是：

> 我想快速变好看 / 遮盖缺陷 / 切换风格，但我担心不自然、闷热、掉落、尴尬、踩雷。

所以机会卡要看完全不同的东西：

| 维度   | 假发类目关注点                            |
| ---- | ---------------------------------- |
| 人群状态 | 发量少、秃顶、发际线高、白发、产后脱发、学生党、coser、职场女性 |
| 变美目标 | 显脸小、显发量、显年轻、显温柔、显贵气、换风格            |
| 使用场景 | 通勤、约会、拍照、婚礼、面试、cosplay、直播、出游       |
| 核心顾虑 | 假不假、热不热、会不会掉、会不会尴尬、会不会显头大          |
| 产品技术 | 刘海片、发片、全头套、递针、蕾丝、头皮仿真、真人发/高温丝      |
| 决策证据 | 真人上头效果、近距离头顶展示、风吹测试、佩戴教程、对比前后      |
| 内容爆点 | 素人改造、秃头救星、妈生感、伪素颜、显脸小、氛围感          |

假发类目机会卡的本质是：

> **某类人，在某个形象焦虑/变美任务下，需要一个自然、安全、可验证的快速形象解决方案。**

---

# 二、假发类目从小红书形成机会卡，建议看 8 个大维度

## 维度 1：人群痛点

假发不是单纯卖“头发”，而是卖“形象解决方案”。

重点识别这些人群：

| 人群          | 典型痛点         | 机会方向            |
| ----------- | ------------ | --------------- |
| 发量少女生       | 头顶塌、发缝宽、拍照显老 | 头顶发片、蓬松发垫、空气感发片 |
| 发际线高        | 扎头发不好看、显脸大   | 发际线修饰片、刘海片      |
| 产后脱发        | 突然掉发、焦虑、自卑   | 自然蓬松发顶片、日常通勤款   |
| 白发人群        | 染发频繁伤发       | 遮白发片、局部补发       |
| 秃顶/脱发男性     | 社交尴尬、显老      | 男士仿真头皮假发        |
| Coser/二次元用户 | 造型还原度、颜色夸张   | cosplay 假发、角色同款 |
| 拍照/直播用户     | 镜头前造型要稳定     | 镜头感假发、上镜发型      |
| 新手用户        | 怕戴不好、怕假      | 新手友好款、懒人假发      |

机会卡不能只写“假发很火”，而要写：

> “产后 3-8 个月女性，因为头顶发缝变宽，在小红书大量搜索‘产后脱发怎么遮’，对‘自然头顶发片 + 真实佩戴教程’有强需求。”

这才是可执行的机会。

---

## 维度 2：场景任务

假发类目要从“商品分类”切到“用户任务”。

| 场景任务    | 用户想完成什么  | 内容机会        |
| ------- | -------- | ----------- |
| 通勤变自然   | 上班戴不尴尬   | 自然光近距离测试    |
| 约会变好看   | 快速提升氛围感  | 妆发穿搭整体改造    |
| 拍照上镜    | 显脸小、显发量  | 前后对比、镜头测试   |
| 婚礼/证件照  | 一次性重要场合  | 场景专用造型      |
| 直播/短视频  | 镜头前稳定好看  | 高光下是否假、掉不掉  |
| 旅行      | 免打理、拍照出片 | 一顶假发多套穿搭    |
| 掉发遮盖    | 解决真实焦虑   | 遮发缝、遮白发、遮秃顶 |
| cosplay | 角色还原     | 角色同款发型      |

机会卡要表达为：

> **人群 × 场景 × 形象目标 × 佩戴顾虑 × 产品解决方案。**

例如：

```yaml
opportunity_title: 通勤女生的“妈生感头顶发片”机会
target_people: 发量少、头顶塌、发缝宽的25-35岁职场女性
scene: 日常通勤 / 办公室 / 地铁自然光
core_job: 不想被看出戴了假发，但希望头顶显蓬松、脸更小
pain_points:
  - 怕头顶反光显假
  - 怕同事近距离看出来
  - 怕佩戴麻烦
content_evidence:
  - 小红书大量出现“发缝宽”“头顶塌”“妈生感发片”“自然到看不出”等表达
product_direction:
  - 轻薄头顶发片
  - 仿真头皮
  - 低密度自然发量
  - 夹子稳固但不扯头发
content_angle:
  - 办公室自然光近距离实测
  - 佩戴前后头顶对比
  - 同事视角能不能看出来
```

---

## 维度 3：情绪与心理障碍

假发类目比桌布更强烈依赖“信任”。

用户不是不知道假发有用，而是害怕：

| 心理障碍 | 内容表达               |
| ---- | ------------------ |
| 怕假   | “近距离也看不出”“头顶自然光实拍” |
| 怕尴尬  | “第一次戴假发上班真实体验”     |
| 怕掉   | “大风天测试”“甩头测试”      |
| 怕闷   | “夏天戴 8 小时会不会热”     |
| 怕头大  | “小头围女生怎么选”         |
| 怕廉价  | “别买一眼假的假发”         |
| 怕不会戴 | “新手 3 分钟佩戴教程”      |
| 怕踩雷  | “假发红黑榜”“不同价位对比”    |

所以假发机会卡一定要加一个字段：

```yaml
trust_barrier:
  - 用户最担心什么
  - 哪些证据能打消顾虑
  - 内容里必须拍什么镜头
  - 商品详情页必须展示什么
```

这是假发类目非常关键的差异点。

---

## 维度 4：内容爆点结构

假发在小红书上容易爆的内容，不只是“产品好看”，而是几个固定结构：

| 爆点结构  | 典型标题              |
| ----- | ----------------- |
| 改造前后  | “戴上假发后我妈都愣住了”     |
| 遮丑救急  | “发缝宽真的别再硬扛了”      |
| 新手避坑  | “第一次买假发一定要看这几点”   |
| 真实测评  | “99 和 599 的假发差在哪” |
| 近距离验证 | “这么近看还假吗？”        |
| 场景挑战  | “戴假发通勤一天会不会翻车”    |
| 人群共鸣  | “产后脱发女生真的太懂了”     |
| 风格切换  | “一顶假发从普通到韩剧女主”    |

机会卡里面不能只记录标题，而要抽象成：

```yaml
content_pattern:
  pattern_type: 前后反差 / 信任验证 / 新手避坑 / 场景挑战 / 人群共鸣
  hook: 第一眼抓人的表达
  proof: 中段提供什么证据
  conversion: 最后如何导向商品
```

---

## 维度 5：商品技术点

假发类目有较强的产品技术门槛。小红书笔记里的用户语言和供应链语言需要映射。

| 用户语言     | 商品/供应链语言          |
| -------- | ----------------- |
| 看起来像自己长的 | 仿真头皮、递针工艺、低密度发量   |
| 不要一眼假    | 发丝光泽控制、发旋处理、头皮分界线 |
| 夏天别太热    | 透气网帽、轻量化底网        |
| 不要掉      | 防滑夹、隐形夹、弹力网帽      |
| 不显头大     | 低蓬松度、头围适配、小头围版型   |
| 好打理      | 高温丝、免打理卷度、可水洗     |
| 遮白发      | 局部发片、混色发丝         |
| 遮发缝      | 头顶补发片、发缝片         |

所以机会卡必须支持：

```yaml
user_expression_to_product_feature:
  user_phrase: "别一眼假"
  product_feature:
    - 仿真头皮
    - 哑光发丝
    - 自然发旋
    - 低密度发量
  proof_content:
    - 头顶近拍
    - 自然光展示
    - 真人佩戴前后
```

这一步就是你们“本体大脑”的价值：把小红书的自然语言，映射成商品策略和内容策略。

---

## 维度 6：人群分层与价格带

假发类目价格带会显著影响机会判断。

| 价格带          | 用户心理              | 机会打法          |
| ------------ | ----------------- | ------------- |
| 低价 50-150    | 尝鲜、拍照、学生党、cosplay | 款式多、上镜、性价比    |
| 中价 150-500   | 日常佩戴、自然度要求较高      | 自然、舒适、新手友好    |
| 高价 500-2000+ | 脱发遮盖、真人发、长期使用     | 信任、专业、售后、定制   |
| 更高价          | 补发、医学级需求          | 专业服务、私域咨询、定制化 |

机会卡要判断：

```yaml
price_band_opportunity:
  current_content_heat: 高
  competition_intensity: 中
  user_trust_threshold: 高
  suitable_product_band: 199-499
  reason: 用户已经有强痛点，但仍需要自然度和佩戴证据支撑
```

---

## 维度 7：证据强度

桌布可以靠审美种草，但假发必须靠证据成交。

假发机会卡建议设置“证据强度评分”：

| 证据类型          | 权重 |
| ------------- | -- |
| 真人佩戴前后对比      | 高  |
| 头顶近距离自然光展示    | 高  |
| 使用一天后的状态      | 高  |
| 用户评论提到“不假/自然” | 高  |
| 风吹/运动/低头测试    | 中高 |
| 多头围/多脸型展示     | 中  |
| 仅棚拍图          | 低  |
| 只有精修模特图       | 低  |

机会卡判断逻辑：

```yaml
evidence_score:
  real_person_before_after: 9
  close_up_scalp_shot: 8
  daily_scene_test: 7
  comment_validation: 8
  total_score: 8.0
```

如果证据分低，即使热度高，也不能直接进入机会池。

---

## 维度 8：内容到商品的可执行性

最终机会卡必须回答：

> 这个机会能不能转成商品、内容、投放和测试任务？

例如：

```yaml
execution_plan:
  product_action:
    - 上架3款自然头顶发片
    - 区分黑茶色/自然黑/棕黑色
    - 增加小头围版本
  content_action:
    - 拍3条“通勤自然光测试”笔记
    - 拍2条“发缝宽前后对比”
    - 拍1条“新手佩戴教程”
  asset_action:
    - 真人前后对比图
    - 头顶近距离图
    - 风吹测试短视频
  test_metric:
    - 笔记收藏率
    - 评论中“自然吗/链接/怎么买”的占比
    - 商品点击率
    - 加购率
```

# 四、如果以后要做多品类，不能为每个类目硬写规则

更好的方式是建立一个 **多品类机会卡生成框架**。

核心架构是：

```text
小红书笔记
  ↓
内容信号抽取
  ↓
类目本体识别
  ↓
用户任务识别
  ↓
痛点/场景/人群/信任障碍提取
  ↓
商品能力映射
  ↓
机会卡生成
  ↓
人工评分/反馈
  ↓
类目透镜持续进化
```

---

# 五、多品类通用的“机会卡五层模型”

建议以后所有类目都用这 5 层。

## 第 1 层：内容信号层

从小红书笔记里抽取：

| 信号      | 示例             |
| ------- | -------------- |
| 热门标题    | “发缝宽女生必看”      |
| 高频关键词   | 发缝宽、妈生感、自然假发   |
| 图片/视频元素 | 前后对比、近距离头顶、自然光 |
| 评论问题    | 会不会假？夏天热吗？     |
| 收藏理由    | 教程、避坑、清单、测评    |
| 情绪词     | 救命、后悔没早买、太自然了  |
| 场景词     | 通勤、约会、拍照、婚礼    |

这是“原始信号”。

---

## 第 2 层：类目本体层

不同类目需要不同的解释框架。

| 类目   | 核心类目本体                     |
| ---- | -------------------------- |
| 桌布   | 空间、风格、材质、节日、尺寸、清洁          |
| 假发   | 人群状态、发型、佩戴方式、自然度、头皮工艺、信任证据 |
| 女装   | 身材、风格、场合、版型、面料、显瘦点         |
| 饰品   | 风格、脸型、穿搭、材质、礼赠场景           |
| 宠物用品 | 宠物类型、行为痛点、安全性、清洁、耐用        |
| 家居收纳 | 空间问题、容量、动线、美观、安装           |
| 美妆   | 肤质、妆效、成分、场景、持久度            |
| 母婴   | 月龄、安全、成长阶段、照护痛点            |

这层决定“看什么”。

---

## 第 3 层：用户任务层

不要问“这个品类有什么趋势”，要问：

> 用户在什么情况下，想完成什么任务？

通用结构：

```yaml
user_job:
  who: 谁
  when: 在什么场景下
  problem: 遇到什么问题
  desired_outcome: 想达到什么结果
  current_alternative: 现在怎么解决
  frustration: 现在方案哪里不好
```

例子：

```yaml
who: 发缝宽的职场女性
when: 每天通勤、办公室社交、拍照
problem: 头顶塌、显老、发量少
desired_outcome: 看起来自然蓬松、不被发现
current_alternative: 用蓬蓬粉、烫发、戴帽子
frustration: 麻烦、不持久、容易尴尬
```

这层决定“为什么买”。

---

## 第 4 层：商品映射层

把小红书洞察转成商品动作。

```yaml
product_mapping:
  user_need: 自然到看不出
  product_features:
    - 仿真头皮
    - 自然发旋
    - 哑光发丝
  sku_action:
    - 增加自然黑/黑茶色
    - 做低密度版本
    - 加小头围选项
  detail_page_action:
    - 增加近距离头顶图
    - 增加真人佩戴视频
```

这层决定“卖什么”。

---

## 第 5 层：内容执行层

把机会卡转成内容任务。

```yaml
content_execution:
```

# 七、机会卡评分公式建议

多品类通用评分可以这样设计：

```text
机会总分 =
需求强度 × 0.25
+ 内容热度 × 0.20
+ 信任缺口 × 0.15
+ 商品匹配度 × 0.20
+ 竞争可切入度 × 0.10
+ 执行可行性 × 0.10
```

但不同类目权重不同。

## 假发类目权重

```yaml
wig_scoring:
  pain_score: 0.25
  trust_gap_score: 0.25
  content_heat_score: 0.15
  product_fit_score: 0.20
  execution_score: 0.10
  competition_gap_score: 0.05
```

假发要提高“痛点”和“信任缺口”的权重。

## 桌布类目权重

```yaml
tablecloth_scoring:
  scene_heat_score: 0.25
  style_trend_score: 0.25
  product_fit_score: 0.20
  content_execution_score: 0.15
  competition_gap_score: 0.10
  trust_gap_score: 0.05
```

桌布要提高“场景”和“风格趋势”的权重。

# 八、系统产品上建议怎么升级

你们原来的内容策划工作台可以升级为：

## 1. 从“单类目机会生成”升级为“多类目机会雷达”

左侧选择：

```text
类目：桌布 / 假发 / 饰品 / 宠物用品 / 家居收纳 / 女装
平台：小红书
目标：找新品机会 / 找内容角度 / 找爆款卖点 / 找人群痛点
```

---

## 2. 中间机会卡看板

每张机会卡包含：

```text
机会标题
人群
场景
痛点
小红书信号
商品方向
内容方向
证据强度
机会评分
推荐动作
```

---

## 3. 右侧 Insight Inspector

点击机会卡后，右侧显示：

```text

4. 对应哪些商品能力？
5. 需要拍什么内容验证？

```

## 4. 底层加入 Category Lens 管理

专门有一个“类目洞察配置”页面：

```text
信任障碍词库

```

这就是以后多品类规模化的关键。

# 十、结论

如果是假发类目，机会洞察的核心不是“哪个发型火”，而是：

> **谁因为哪种形象问题产生焦虑，在什么场景下需要快速变好看，他最担心什么，什么产品特征和内容证据能打消顾虑。**

如果未来做多品类，核心能力也不是“写更多 prompt”，而是建立：

> **Category Lens 类目透镜 + 通用机会卡 Schema + 类目化评分权重 + 人工反馈回流机制。**

这样系统才能从“桌布内容策划工具”升级成真正的：

> **多品类小红书机会发现与内容策划引擎。**
